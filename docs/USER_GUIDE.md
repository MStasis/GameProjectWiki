# 사용자 안내서

## 1. 설치 전 확인

이 위키는 중앙 클라우드 서버가 없는 개인용 앱입니다. PC와 휴대폰이 각각 전체 위키를 갖고, 집의 PC가 켜져 있을 때 변경 사항을 합칩니다.

준비물은 다음과 같습니다.

- Windows 10/11 x64 PC
- Android 휴대폰
- 두 기기에서 사용할 같은 Tailscale 계정
- GitHub Release의 Windows EXE와 Android APK

설치 파일과 Release에 적힌 SHA-256 값이 일치하는지 확인할 수 있습니다.

```powershell
Get-FileHash .\Title-Placeholder-Wiki-Setup-2.0.0.exe -Algorithm SHA256
```

Android 파일도 PC에 내려받았다면 같은 명령으로 확인할 수 있습니다.

## 2. Windows 앱 설치

1. `Title-Placeholder-Wiki-Setup-2.0.0.exe`를 실행합니다.
2. SmartScreen이 “Windows의 PC 보호”를 표시하면 게시자와 파일 해시를 다시 확인합니다.
3. 공식 Release 파일이 맞을 때만 **추가 정보 → 실행**을 선택합니다.
4. 설치 위치와 바로 가기를 선택해 설치합니다.
5. Title Placeholder Wiki를 실행합니다.

이 EXE는 악성 파일이어서가 아니라 유료 코드 서명 인증서로 서명하지 않았기 때문에 SmartScreen 평판 경고가 날 수 있습니다. 출처나 해시가 다르면 실행하지 마세요.

PC 앱의 닫기 버튼은 창만 숨깁니다. 시스템 트레이에 앱이 남아 있어야 휴대폰이 동기화할 수 있습니다. 앱을 완전히 끝내려면 트레이 아이콘을 우클릭하고 **종료**를 선택합니다.

첫 실행 후 **설정 및 동기화 → Windows 로그인 시 자동 시작**을 켜 두면 다음 로그인부터 창을 띄우지 않고 트레이에서 동기화 호스트가 준비됩니다. 휴대폰 변경을 집에 돌아온 뒤 자동으로 합치려면 이 옵션을 권장합니다.

## 3. Android APK 설치

1. `Title-Placeholder-Wiki-Android-v2.0.0.apk`를 휴대폰에 내려받습니다.
2. 파일 관리자나 브라우저에서 APK를 엽니다.
3. Android가 요청하면 이번에 사용한 앱에 한해서 **알 수 없는 앱 설치 허용**을 켭니다.
4. 설치를 완료한 뒤 해당 허용을 다시 꺼도 됩니다.

APK는 프로젝트의 개인 릴리스 키로 서명되지만 Play 스토어에서 배포하지 않으므로 Android와 Play Protect가 경고할 수 있습니다. 공식 Release와 SHA-256 값을 확인한 파일만 설치하세요. 업데이트 APK는 기존 버전과 같은 릴리스 키로 서명되어야 데이터와 앱을 유지한 채 설치할 수 있습니다.

## 4. Tailscale로 두 기기 연결

Tailscale은 PC 동기화 주소를 인터넷 전체에 공개하지 않고, 본인의 기기만 참여하는 사설 네트워크로 연결합니다.

1. PC와 휴대폰에 [Tailscale](https://tailscale.com/download)을 설치합니다.
2. 두 기기에서 같은 계정으로 로그인하고 연결 상태가 **Connected**인지 확인합니다.
3. PC 위키 앱의 **설정/동기화** 화면에서 Tailscale Serve를 활성화합니다.
4. 성공하면 다음 정보가 PC에 표시됩니다.
   - `https://<PC이름>.<tailnet이름>.ts.net` 형태의 주소
   - 동기화 사용자명
   - 자동 생성된 긴 비밀번호
5. 휴대폰 위키의 **설정/동기화 서버**에 세 값을 그대로 입력하고 저장합니다.
6. **연결 확인** 또는 **지금 동기화**를 실행합니다.

화면의 메뉴 명칭은 앱 버전에 따라 조금 다를 수 있지만, 입력해야 할 값은 URL·사용자명·비밀번호 세 가지입니다. URL 끝에 임의의 경로나 포트를 추가하지 마세요.

자격 증명은 위키 전체를 읽고 쓸 수 있는 열쇠입니다. 메신저로 전송하거나 스크린샷을 공유하지 마세요. PC와 휴대폰의 로컬 설정에만 보관하는 것이 좋습니다.

### 연결되지 않을 때

- PC 위키 앱이 실행 중이거나 트레이에 남아 있는지 확인합니다.
- 두 기기의 Tailscale이 같은 계정과 같은 tailnet에 연결됐는지 확인합니다.
- PC 설정에서 Tailscale Serve 활성화를 다시 실행합니다.
- 휴대폰 주소의 `https://`, 사용자명, 비밀번호에 공백이나 오타가 없는지 확인합니다.
- 모바일 데이터와 Wi-Fi를 바꿔 본 뒤 Tailscale 연결 상태를 다시 확인합니다.
- Windows 방화벽을 전체 공개로 풀지 마세요. 앱은 로컬 포트 `8765`를 사용하고 외부 연결은 Tailscale Serve를 통합니다.

## 5. 오프라인 편집과 동기화

각 기기의 문서와 이미지는 해당 기기의 로컬 데이터베이스에 먼저 저장됩니다. 따라서 지하철이나 야외처럼 인터넷이 없거나 집 PC가 꺼진 상황에서도 휴대폰에서 작성·수정·삭제할 수 있습니다.

PC가 다시 켜지고 연결되면 다음 순서로 동기화됩니다.

1. 휴대폰의 대기 중인 변경이 PC 동기화 호스트로 전송됩니다.
2. PC에만 있던 변경이 휴대폰으로 내려옵니다.
3. 처리된 변경은 대기 목록에서 제거됩니다.
4. 같은 문서를 양쪽에서 수정했다면 덮어쓰지 않고 충돌로 보관합니다.

앱이 열린 상태에서는 자동 동기화를 시도하고, 언제든 상단 상태 버튼이나 설정의 **지금 동기화**로 즉시 실행할 수 있습니다. Android가 앱을 완전히 종료한 상태에서는 백그라운드 동기화를 보장하지 않으므로, 집에 돌아온 뒤 앱을 열고 한 번 동기화하는 습관을 권장합니다.

## 6. 위키 작성

### 주제와 문서

- 폴더는 대주제와 하위 주제에 사용합니다. 예: `무기 → AR → AR-17`.
- 문서는 실제 아이템이나 시스템의 정보를 담습니다.
- 폴더 안에 폴더를 다시 만들어 필요한 깊이까지 분류할 수 있습니다.
- 구조 관리 화면에서 데스크톱은 드래그 앤 드롭, 모바일은 위/아래·들여쓰기·내어쓰기 버튼으로 순서를 바꿉니다.
- 삭제된 항목은 먼저 휴지통으로 이동하므로 복원할 수 있습니다.

### 콘텐츠 블록

문서는 다음 블록을 원하는 순서로 조합합니다.

- **텍스트:** 글꼴, 크기, 굵게, 기울임, 밑줄, 취소선, 정렬, 목록, 글자색, 배경색
- **이미지:** PNG, JPEG, WebP, GIF와 대체 텍스트·설명. 이미지는 위키 데이터에 포함되어 기기 사이에 동기화됩니다.
- **YouTube:** 일반 영상 또는 공유 URL. 영상 플레이어는 인터넷에 연결됐을 때만 재생됩니다.
- **Google Sheets:** 스프레드시트 링크와 `WeaponData!A1:H24` 같은 범위
- **강조 상자:** 정보, 주의, 위험, 완료 같은 안내
- **구분선:** 긴 문서의 영역 분리

블록 역시 데스크톱에서는 드래그하고 모바일에서는 이동 버튼을 사용해 순서를 바꿀 수 있습니다.

### Google Sheets 공개 범위

Sheets 블록은 Google이 iframe으로 보여 줄 수 있는 공개 문서가 필요합니다. Google Sheets에서 공유 또는 웹 게시 범위를 먼저 설정한 뒤 링크와 표시 범위를 입력하세요.

- 링크를 가진 모든 사람이 볼 수 있도록 공개한 데이터는 이 앱의 보안 범위 밖입니다.
- 비밀번호, 개인 정보, 비공개 밸런스 자료는 공개 시트에 넣지 마세요.
- 시트의 권한을 나중에 비공개로 바꾸면 위키 블록에서도 표시되지 않습니다.
- 시트 내용은 위키에 복사되어 저장되는 것이 아니므로 오프라인에서는 불러올 수 없습니다.

## 7. 충돌 해결

PC가 꺼져 있는 동안 휴대폰에서 한 문서를 수정하고, 같은 시기에 PC에서도 이전 사본을 수정하면 두 변경 중 하나를 임의로 버리지 않습니다. 동기화 센터의 **충돌** 목록에 현재 사본과 들어온 사본을 함께 보관합니다.

1. 설정 또는 동기화 센터에서 충돌 목록을 엽니다.
2. 문서 제목, 기기, 수정 시각과 양쪽 내용을 비교합니다.
3. PC 쪽 유지, 휴대폰 쪽 유지, 또는 필요한 내용을 합친 새 사본 중 하나를 선택합니다.
4. 해결 내용을 저장하고 **지금 동기화**를 다시 실행합니다.

충돌을 해결하기 전에도 다른 문서는 계속 사용할 수 있습니다. 다만 충돌한 문서를 양쪽에서 계속 편집하면 판단해야 할 버전이 늘 수 있으므로 먼저 정리하는 것이 좋습니다.

## 8. 백업과 복구

### 일반 백업

설정의 **데이터 내보내기/JSON 내보내기**로 위키 전체 사본을 만듭니다. 내보낸 JSON에는 문서, 구조, 블록과 이미지가 포함될 수 있으므로 비밀번호만큼 조심해서 보관하세요. 중요한 변경 전과 앱 업데이트 전에 내보내기를 권장합니다.

PC 앱은 추가로 시작할 때, 하루 간격, 그리고 트레이의 **지금 백업**을 실행할 때 압축 백업을 만듭니다.

```text
내 문서\Title Placeholder Wiki\Backups\wiki-<날짜와 시각>.wiki-backup.json.gz
```

최신 30개를 유지합니다. 실제 경로는 PC 앱 설정에도 표시됩니다. 자동 백업은 일반 백업과 같은 형식의 gzip 압축 파일이므로, 설정의 **백업 파일 가져오기**에서 `.wiki-backup.json.gz` 파일을 그대로 선택해 복구할 수 있습니다. 실행 중인 내부 데이터 파일 위에 직접 덮어쓰지는 마세요.

### 일반 복구

1. 가능하면 현재 상태도 먼저 JSON으로 내보냅니다.
2. 휴대폰과 다른 기기의 자동 동기화를 잠시 멈춥니다.
3. 설정에서 **데이터 가져오기/JSON 가져오기**를 선택합니다.
4. 복구할 JSON과 앱이 보여 주는 문서 수·날짜를 확인한 뒤 적용합니다.
5. 앱을 다시 열고 문서와 이미지가 정상인지 확인합니다.
6. 다른 기기를 차례로 연결해 동기화하고, 발생한 충돌을 검토합니다.

가져오기는 기존 변경과 충돌할 수 있으므로 파일을 확인하지 않은 채 반복 실행하지 마세요. 자동 `.wiki-backup.json.gz`도 동일한 가져오기 화면에서 사용할 수 있으며, 먼저 **현재 기록과 합치기**로 확인하는 것이 안전합니다.

### 기기 데이터 삭제 주의

Android의 **앱 데이터 지우기/저장 공간 삭제**, 앱 제거, 브라우저 저장소 삭제는 그 기기의 로컬 사본을 지웁니다. 먼저 동기화 상태가 완료인지 확인하고 JSON을 내보내세요. PC 앱의 데이터 폴더도 설정에 표시되며, 앱 실행 중에 직접 옮기거나 편집하면 안 됩니다.

## 9. 로컬 보안

- 별도의 앱 회원 계정은 없습니다. 앱을 열 수 있는 Windows/Android 사용자에게는 편집 권한도 있습니다.
- Windows 로그인 암호, BitLocker 또는 장치 암호화, Android 화면 잠금을 권장합니다.
- 동기화 API는 Tailscale의 HTTPS와 별도 사용자명/비밀번호로 보호됩니다.
- 동기화 비밀번호와 데이터 파일은 Git, 이메일, 공개 클라우드 폴더에 올리지 마세요.
- 중고 판매나 기기 폐기 전에는 앱 데이터, 백업과 Tailscale 기기 등록을 제거하세요.
- YouTube와 Google Sheets 블록을 열면 해당 서비스로 네트워크 요청이 전송됩니다.

## 10. 현재 제한

- 설치 패키지는 Windows x64와 Android만 제공합니다.
- iOS 앱은 제공하지 않습니다.
- PC가 꺼진 동안 휴대폰의 편집은 가능하지만 다른 기기로 즉시 전송되지는 않습니다.
- Android 앱이 종료된 동안의 백그라운드 동기화는 보장하지 않습니다.
- YouTube 영상과 Google Sheets는 온라인일 때만 실제 내용을 표시합니다.
- 대용량 이미지를 많이 넣으면 기기 저장 공간과 첫 동기화 시간이 크게 늘 수 있습니다.
- 이 버전은 앱 데이터 자체를 GitHub Pages 같은 공개 사이트에 자동 게시하지 않습니다.
